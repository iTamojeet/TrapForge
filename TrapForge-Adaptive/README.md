# TrapForge Adaptive Response Engine

## Objective

The Adaptive Response Engine dynamically modifies honeypot behaviour according to the Attacker Engagement Index (AEI) generated by the AEI Engine.

## Working

1. The engine continuously reads attacker AEI scores from the `session_scores` database table.
2. Based on predefined thresholds, an appropriate deception level is selected.
3. The selected configuration is written to `current_mode.json`.
4. Honeypot services can read this configuration file and adapt their behaviour.

## Adaptation Rules

### Low AEI (0–3)

* Mode: BORING
* Services: SSH only
* Fake Login: Disabled
* Fake Files: None

Behaviour:
The attacker is presented with a minimal, non-responsive environment to encourage disengagement.

### Medium AEI (4–6)

* Mode: ENGAGING
* Services: SSH, FTP
* Fake Login: Disabled
* Fake Files:

  * employees.csv
  * projects.docx

Behaviour:
Additional fake assets and services are exposed to maintain attacker engagement.

### High AEI (7–10)

* Mode: REALISTIC
* Services: SSH, FTP, HTTP
* Fake Login: Enabled
* Fake Files:

  * salary_data.xlsx
  * financial_report.pdf
  * admin_credentials.txt

Behaviour:
A near-real environment is presented with realistic fake credentials and sensitive-looking information.

## Real-Time Monitoring

The engine continuously monitors AEI values every 5 seconds and dynamically updates the active honeypot configuration.

## Output

The engine updates `current_mode.json`, which can be consumed by honeypot services such as Flask or Cowrie to implement adaptive deception.
